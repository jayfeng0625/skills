# Shopping Cart Module

## Problem/Feature Description

A small e-commerce startup is building the core of their platform. They need a shopping cart module that allows users to add products, update quantities, remove items, and calculate a total with applicable discounts. The team has decided to use test-driven development to build this from scratch, and you have been asked to produce the implementation.

The module does not need to persist data or connect to any external services — it operates entirely in memory. The team wants the implementation to be clean enough that future engineers can easily understand what the system does from the tests alone, without needing to read the implementation.

## Output Specification

Produce the following files in your working directory:

- `cart.js` (or equivalent in a language of your choice) — the shopping cart implementation
- `cart.test.js` (or equivalent) — the test suite covering cart behaviors
- `tdd-log.md` — a short log documenting the order in which you wrote tests and implemented code, one entry per RED→GREEN cycle

The test file and implementation should be self-contained and runnable without external dependencies beyond the standard testing library for your chosen language.
