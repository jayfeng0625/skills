# Technical Explainer — Caveman Mode

## Problem/Feature Description

A backend developer is getting up to speed on some core infrastructure concepts. They've asked for explanations of three topics that will be relevant to their new role. They've explicitly asked you to talk like a caveman — they want dense, efficient answers with no fluff.

Explain each of the following three concepts:

1. How connection pooling works and why it matters under load
2. Why JWT tokens can be hard to revoke and what alternatives exist
3. How database index scans differ from full table scans and when each happens

## Output Specification

Write your explanations to a file called `explanations.md`. Address each topic in order. Use caveman mode throughout — terse, compressed, technically accurate.
