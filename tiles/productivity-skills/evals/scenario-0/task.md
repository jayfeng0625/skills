# Social Media Scheduling Skill

## Problem/Feature Description

A small marketing team manages social media presence across several platforms for multiple clients. Right now, each team member keeps their own notes in scattered documents about how to schedule posts — what times work best, what tools to use, which clients need approval before publishing. The inconsistency is causing mistakes: posts go out at wrong times, platform-specific formatting is missed, and new team members take weeks to get up to speed.

The team lead wants to capture this knowledge in a formal skill that agents can load on demand to help with social media scheduling tasks. The skill should be broadly useful: it should cover drafting and scheduling posts, understanding platform constraints (character limits, image ratios, hashtag conventions), and knowing when to seek approval. It doesn't need to connect to any external APIs — it should provide guidance and a workflow.

## Output Specification

Create a skill directory called `social-media-scheduler/` containing all the files needed for a working agent skill. The skill should be immediately usable by another agent who loads it.

Write a file called `skill-report.md` that briefly explains what files you created and why you made the structural choices you did.
