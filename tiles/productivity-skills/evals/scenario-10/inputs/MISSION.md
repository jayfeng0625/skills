# Mission: Understanding async/await in JavaScript

## Why I want to learn this

I'm a JavaScript developer comfortable with callbacks and promises, but I struggle when async/await is combined with error handling and parallel execution. I frequently write code that works but then breaks in edge cases I didn't anticipate.

## Success looks like

- I can explain what `async` and `await` do mechanically (not just syntactic sugar)
- I can write async functions that handle errors correctly with try/catch
- I can run multiple async operations in parallel using Promise.all and understand when to use it
- I can read unfamiliar async code and predict its execution order

## Prior knowledge

Strong JavaScript fundamentals — functions, closures, the event loop, Promise basics. Weak on: Promise chaining edge cases, error propagation across await boundaries.
