# Skill Quality Improvement

## Problem/Feature Description

A developer has written their first agent skill and wants a peer review before rolling it out to the team. The skill is meant to help with capturing and structuring meeting notes. The file is at `inputs/SKILL.md`.

They're not sure if it follows best practices for skill authoring. Some team members have said it seems hard to know when to load the skill, and one reported finding outdated information in it. The developer wants an expert to go through it carefully and produce an improved version that addresses any structural or quality issues.

Your job is to audit the provided skill and rewrite it so it is production-ready. Do not change the core purpose of the skill — it should still be about meeting notes — but fix any problems you find with how it is written, structured, or what information it contains.

## Output Specification

Write the improved skill to `meeting-notes/SKILL.md` (create the full skill directory). If additional files are warranted by the improvements, create them in the same directory.

Write a `review-report.md` that lists each problem you found in the original and how you fixed it.
